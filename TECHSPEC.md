# ТЕХНИЧЕСКОЕ ЗАДАНИЕ: Платформа AI-агентов для Telegram (SaaS)

## 1. Архитектура (Monorepo Services)
Проект структурирован как набор микросервисов в директории `services/`, управляемых через `docker-compose.yml`.
- **api:** FastAPI (Прием вебхуков, внутреннее API).
- **bot:** aiogram 3.x (Логика общения с клиентами).
- **admin:** Streamlit (Дашборд владельца: настройки, товары, аналитика).
- **worker:** Celery/ARQ (Фоновые задачи: анализ сессий, алерты).

## 2. Технологический стек
- **Language:** Python 3.12.
- **Package Manager:** **uv** (fast & modern).
- **Testing:** **pytest** + pytest-asyncio.
- **Linting/Formatting:** **Ruff**.
- **Database:** PostgreSQL 16.
- **Search:** PostgreSQL Full Text Search (индексы `tsvector`) + JSONB (динамические атрибуты).
- **LLM:** OpenRouter (GPT-4o / Claude 3.5 Sonnet).

## 3. Модель данных (SQLAlchemy 2.0 Mapped)
- `Tenant`: Настройки магазина (`bot_token`, `system_prompt`, `catalog_schema`).
- `Product`: Каталог товаров (FTS индексы по `name` и `description`).
- `ChatSession`: Управление сессиями (аналитика запускается после закрытия сессии).
- `Message`: История для контекста.
- `Order`: Предзаказы для сборщиков.

## 4. Ключевая логика
### 4.1. Поиск (Hybrid SQL)
Вместо векторов используем комбинацию:
- `to_tsquery` для смыслового поиска по тексту.
- JSONB операторы (`->>`, `@>`) для фильтрации по динамическим характеристикам.

### 4.2. Аналитика диалогов
После завершения сессии (2 часа неактивности) LLM анализирует историю на предмет:
- Интента и удовлетворенности клиента.
- Упущенной выгоды (чего не было в наличии).
- Советов по улучшению бизнеса.

## 5. Инфраструктура
- Развертывание через `docker-compose`.
- Использование `Makefile` для автоматизации команд (build, dev, test).
